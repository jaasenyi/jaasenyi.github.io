Java面向对象

1、什么是面向对象

  • 面向过程思想

    • 步骤清晰，第一步做什么，第二步做什么……

  • 面向对象思想：

    • 分类的思维模式，思考解决问题需要哪些分类，对分类进行单独思考，最后对某个分类下的细节进行面向过程思索。

  • 对于描述复杂的事物，为了从宏观把握，整体分析，需要使用面向对象的思路；

    具体到微观操作，仍需面向过程思路处理。

  • 面向对象编程的本质：以类的方式组织代码，以对象的形式组织（封装）数据。

  • 三大特性

    • 封装
    • 继承
    • 多态

2、类与对象的关系

  1. 类是一种抽象的数据类型，对某一类事物整体描述，不能代表某一具体事物。
    • 静态的属性-属性
    • 动态的行为-方法
  2. 对象是抽象概念的具体实例，类是对象的模板。

（1）使用new关键字创建对象：

  • 分配内存空间。
  • 给创建好的对象默认初始化。
  • 对类中构造器调用。

（2）构造器

类中构造器也称构造方法，是在进行创建对象时候必须要调用的。特点：

  • 必须和类的名字相同
  • 必须没有返回类型，也不能写 void
//Student类
public class Student {
    //一个类什么都不写，也会存在一个方法；
    //显示的定义构造器
    String name;
    
    //1、使用new关键字，本身是在调用构造器
  	//2、用来初始化值
    public Student(){
        this.name="xiaohong";
    }

    //有参构造：一旦定义有参构造（若还使用无参构造new 对象），无参必须显示定义
    public Student(String name){
        this.name=name;	//this.name当前类中的name
    }
}

/*测试------------------------------------------
public class Demo06 {
    public static void main(String[] args) {
        //类实例化后返回一个自己的对象
        Student student = new Student("xiaoming");
        System.out.println(student.name);
    }
}
//结果：
xiaoming
*/

3、封装

高聚合：类的内部数据操作细节自己完成，不允许外部干涉

低耦合：仅限少量的方法给外部使用

封装：通常，应禁止直接访问一个对象中数据的实际表示，而应通过操作接口来访问。

属性私有，get/set

封装意义：

  • 提高程序的安全性，保护数据
  • 隐藏代码的实现细节
  • 统一接口
  • 系统可持续性增加
public class Student {
    //属性私有
    private String name;
    private char sex;
    private int age;

    //提供一些可操作这些属性的方法！
    //提供一些public的get、set方法

    //get获取这个数据
    public String getName() {
        return this.name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return this.age;
    }

    public void setAge(int age) {
        if (age>120||age<0){
            this.age = 3;
        }else {
            this.age = age;
        }
    }
}
/*测试-----------------------------------------
public class Demo07 {
    public static void main(String[] args) {
        Student s1 = new Student();

        s1.setName("小明");
        s1.setAge(-1); //不合法

        System.out.println(s1.getName());
        System.out.println(s1.getAge());
    }
}
//结果：
小明
3
*/

4、继承

继承是类和类之间的关系，除此之外类和类之间还有依赖、组合、聚合等；

继承关系是两个类，一个是子类（派生类），一个是父类（基类）。子类继承父类，使用extends来表示。

JAVA中类只有单继承，没有多继承！

//子类继承父类，就会拥有父类全部方法！
public class Student extends Person{}
//Java中，所有的类都默认直接或间接继承Object类
//父类中私有(private)属性不会继承

（1）Super

  • super调用父类的构造方法，必须在构造方法的第一个
  • super必须只能出现在子类的方法或构造方法中！
  • super和this不能同时调用构造方法
This Super
本身调用这个对象 代表父类对象的引用
没有继承也可以使用 只能在继承条件下才可以使用
默认调用本类的构造 调用父类的构造 
//Persion.java
public class Persion {
    protected String name = "persion";
}
//--------------------------------------------
//Student.java
public class Student extends Persion{

    private String name = "student";

    public void test(String name){
        System.out.println(name);
        System.out.println(this.name); //student
        System.out.println(super.name);//结果persion;子类调用父类的属性super.name
    }
}

import com.jaa.method.testclass.*;

       Student s1 = new Student();
       s1.test("xiaoming");

    }
}

（2）重写方法

需要有继承关系，子类重写父类的方法！

  • 方法名必须相同
  • 参数列表必须相同
  • 修饰符：范围可以扩大：public > protected > default > private
  • 抛出的异常：范围可以被缩小，但不能扩大
/*补充：
父类和子类方法名与参数列表相同时：
1、静态方法：方法的调用只和左边，定义的数据类型有关
2、非静态方法：子类重写父类方法，执行子类。（重写）
*/

5、多态

同一方法可以根据发送对象的不同而采用多种不同的行为方式

//Persion.java
public class Persion {
    public void run(){
        System.out.println("run");
    }

}
//------------------------------------------
//Student.java
public class Student extends Persion{

    public void run() {
        System.out.println("son");
    }
    public void eat(){
        System.out.println("eat");
    }
}
//--------------------------------------------
//测试：
ublic class Demo09 {
    public static void main(String[] args) {
        //一个对象的实际类型是确定的
        //new Student()
        //new Persion()

        //可以指向的引用类型不确定:父类的引用指向子类的类型
        Student s1 = new Student();
        Persion s2 = new Student();

        //对象能执行那些方法看对象左边的类型和右边关系不大！
        s1.eat();
       // s2.eat(); 无法运行，父类中无此方法
    }
}

多态注意事项：

  • 多态是方法的多态，属性没有多态！
  • 存在于父类和子类关系中；类型转换异常！ClassCastException!
  • 存在条件：继承关系，方法需要重写，父类的引用指向子类对象！
  • 不能重写的方法：
    • static 方法，属于类，不属于实例
    • final 常量
    • private方法

6、抽象类、接口

（1）抽象类

  • **abstract**修饰的类、方法为抽象类、抽象方法
  • 继承了抽象类的子类都必须要实现他的方法
//抽象类
public abstract class action {
    //抽象方法，只有方法的名字，没有方法的实现
    public abstract void doSomthings();

    //1、抽象类不能new，只能靠子类去实现！
    //2、抽象类中可以写普通方法
    //3、抽象方法必须在抽象类中
}

（2）接口

普通类 抽象类 接口
只有具体实现 具体实现和规范（抽象方法）都有 只有规范！自己无法写方法
  • 接口：只有规范！自己无法写方法～约束和实现分离：面向接口编程～
  • 接口不能被实例化，接口中没有构造方法
  • 接口的本质是契约
  • 实现了接口的类，必须重写接口的所有的方法
  • implements利用接口实现多继承
//interface定义关键字，接口都需要有实现类
public interface UserService {
    //接口中所有的定义的方法都是抽象的 public abstract
    //接口中定义的常量 public static final
    void add(String name);
    void del(String name);
    void update(String name);
    void sel(String name); 
}
public interface TimeService {
    void timer();
}
//类 可以实现接口 implements 接口
//实现了接口的类，必须重写接口的所有的方法
//利用接口实现多继承
public class UserServiceImpl implements UserService,TimeService{

    @Override
    public void add(String name) {

    }

    @Override
    public void del(String name) {

    }

    @Override
    public void update(String name) {

    }

    @Override
    public void sel(String name) {

    }

    @Override
    public void timer() {

    }
}

8、异常

（1）什么事异常

检查性异常

运行时异常

错误ERROR

Java把异常当作对象来处理，并定义一个基类**java.lang.Throwable作为所有异常的超类**

Java API中已经定义了很多异常类，这些异常类分为两大类错误Error和异常Exception

  • Exception分支中有一个重要的子类RuntimeException(运行时异常)
异常： 解释：
ArrayIndexOutOfBoundsException 数组下标越界
NullPointerException 空指针异常
ArithmeticException 算数异常
MissingresourceException 丢失资源
ClassNotFoundException 找不到类

这些异常时不检查异常，程序中可以选择捕获处理，也可以不处理。

Error Exception
通常为灾难性的致命错误，程序无法控制和处理，当出现这些异常时，**Java虚拟机(JVM)**一般会终止线程； 通常情况下可以被程序处理，并在程序中应该尽可能的被处理的异常

（2）异常处理机制

  • 异常处理五个关键字
  • try catch finally throw throws
public class test {
    public static void main(String[] args) {
        int a=1;
        int b=0;
        try {//监控区域
            System.out.println(a/b);
        }catch (ArithmeticException e){//捕获异常
            System.out.println("程序出现异常，变量b不能为0");
        }finally {//处理善后工作（无论异常是否出现，都会执行）
            System.out.println("finally");
        }
    }
}
  • 要捕获多个异常，从小到大去捕获
try {//监控区域
            System.out.println(a/b);
        }catch (Error e){//捕获异常
            System.out.println("Error");
        }catch (Exception f){
            System.out.println("Error");
        }catch (Throwable g){
            System.out.println("Throwable");
        }
  • 主动抛出异常
//假设方法中处理不了这个异常。方法上抛出异常
public void test(int a,int b)throws ArithmeticException{
    if (b==0){
        throw new ArithmeticException();//主动抛出异常，一般在方法中使用
    }
}
  • 经验：
    • 处理运行时异常时，采用逻辑去合理规避同时辅助try-catch处理
    • 多重catch块后面，可以加一个catch（Exception）来处理可能会被遗漏的异常
    • 不确定的代码也可加try-catch，处理潜在异常
